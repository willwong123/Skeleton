Skeleton 适合我吗?

如果你在做一个很小的项目或者觉得用不到那些庞大的UI框架的全部功能的话，就可以使用Skeleton。Skeleton 仅设定了少量标准HTML元素的样式和一个网格，不过这绝不是简单的入门。这个网站就是用的Skeleton和一个不超过200行的Custom CSS做的（其中一半还是设置导航的）

喜欢就给个star吧！

网格

网格的最大宽度960px，分成12等份，最大宽度会根据设备/浏览器的尺寸进行缩放，也可以自己在CSS中定义一个最大宽度。它的语法非常简单，使得开发响应式更加的容易。这里可以试着改变浏览器窗口大小看看效果。

段落

段落文字都是用的rems，因此字体大小会根据<html>的字体大小自适应。<html>的字体大小，可以根据项目具体情况设置大小。所有的字体大小基数都是10，<h1> 设置为 5.0rem 算下来就是 50px。

段落样式是基于谷歌开发的Raleway字体，默认是1.5rem(15px)大小，1.6倍行间距。此外还包含一下几种样式链接, 加粗, 强调 和 下划线。

标题使用了另一种特殊的字间距，行高，外边距。

译者注：由于某些原因，国内无法使用谷歌字体，所以需要自行设置字体。

按钮

Skeleton 中设置了两种样式的按钮。默认样式的按钮是用.button样式生成，可以用.button-primary生成一个比较突出的按钮样式。这些按钮样式可以用在input，button元素上，input支持的类型包括[type=submit]，[type=button]，[type=reset]，同样可以用在a上，只是需要添加一个.button的class。

媒体查询

Skeleton使用了媒体查询来保证网格的可扩展性，同样也提供了一些媒体查询来方便实现项目网站的响应式，这里依然遵循移动优先法则，使用 min-width来查询。Skeleton就是在移动优先法则上进行构建组织CSS代码的，也就是说媒体查询之外的代码都是给大屏设备用的，可以保证更好的展现；同时也保证在较小屏幕上CSS文件的纯净。

  • Desktop HD: 1200px
  • Desktop: 1000px
  • Tablet: 750px
  • Phablet: 550px
  • Mobile: 400px
/* 移动优先 */

/* Larger than mobile */
@media (min-width: 400px) {}

/* Larger than phablet */
@media (min-width: 550px) {}

/* Larger than tablet */
@media (min-width: 750px) {}

/* Larger than desktop */
@media (min-width: 1000px) {}

/* Larger than Desktop HD */
@media (min-width: 1200px) {}
公共样式

Skeleton 有一些很小但很好用的公共样式。比如说，浮动一个元素直接使用u-pull-left可以左浮动，而不需要重新编写一个class。

/* 公共样式 */

/* 设置宽度为100% */
.u-full-width {
  width: 100%;
  box-sizing: border-box; }

/* 保证元素不会超出父容器（对columns中的img元素有奇效） */
.u-max-full-width {
  max-width: 100%;
  box-sizing: border-box; }

/* 浮动 */
.u-pull-right {
  float: right; }
.u-pull-left {
  float: left; }

/* 清除浮动 */
.u-cf {
  content: "";
  display: table;
  clear: both; }
